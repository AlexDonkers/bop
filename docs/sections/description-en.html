SPARQL

Querying static and dynamic properties

PREFIX bop: <https://w3id.org/bop#>
PREFIX bot: <https://w3id.org/bot#>

SELECT ?room ?property ?result

WHERE {
?room a bop:FeatureOfInterest, bot:Space .
?room bop:hasProperty ?property .
?property bop:hasPropertyState ?result .
}

Querying the spatiotemporal resolution of a temperature measurement

PREFIX bop: <https://w3id.org/bop#>
PREFIX bot: <https://w3id.org/bot#>
PREFIX quantitykind: <http://qudt.org/2.1/vocab/quantitykind>

SELECT ?host ?sensor ?viewpoint ?relationship ?currentState

WHERE {
?sensor a bop:Sensor .
?sensor bop:observes ?temperature .
?temperature a quantitykind:Temperature .
?temperature bop:hasPropertyState ?currentState .
?sensor bop:isHostedBy ?host .
?temperature bop:isPropertyOf ?viewpoint .
?host ?relationship ?viewpoint .
}

Querying a property and its related executors

PREFIX bop: <https://w3id.org/bop#>
PREFIX bot: <https://w3id.org/bot#>

SELECT ?room ?property ?executor

WHERE {
?room a bop:FeatureOfInterest, bot:Space .
?room bop:hasProperty ?property .
?executor bop:executesOn ?property .
}

Open Smart Home

Open Smart Home

The Open Smart Home is an initiative by Schneider and Rasmussen. The building represents an apartment in Nuremberg, Germany. We converted the revit file to DAE and GLTF formats, and manually created a TTL file of the building that follows the BOP-ontology.

Various sensors measured the temperature, relative humidity, brightness and outdoor temperature for a few months. This data is available in CSV format.

We collected all ontologies that were used in the TTL file of the Open Smart Home and made them available in the downloads section.
